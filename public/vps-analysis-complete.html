‚úÖ Validasi Metodologi Riset

Yang Sudah Benar

1. Fokus pada 4K Random IOPS

Pemilihan 4K random IOPS sebagai metrik utama adalah SANGAT TEPAT. Chrome automation menghasilkan pola akses storage yang intensive: menulis cache files, temporary files, session data, cookies, localStorage, IndexedDB secara konstan dalam ukuran block kecil.

Netcup RS 1000 G12: ~93,000 IOPS
Contabo VPS: ~2,600 IOPS
Perbedaan: 35x lebih cepat!

üí° IOPS rendah akan membuat Chrome lambat, tiap buka halaman dan caching akan bottleneck, leading to timeouts dan crashes.

2. Analisis CPU Throttling Policy

Fokus pada sustained performance vs burst performance adalah krusial untuk workload 24/7. Identifikasi kebijakan Hetzner ~20-33% sustained sangat berharga dan AKURAT.

Series Nominal vCPU Sustained % Effective Cores
Hetzner CX43 8 vCPU ~20% ~1.6 cores
Hetzner CPX32 4 vCPU ~33% ~1.3 cores
Netcup RS 1000 4 cores 100% 4.0 cores

‚úÖ Distingsi antara burst vs sustained adalah scientifically sound untuk 24/7 automation.

3. Netcup RS 1000 G12 = Tercepat

Klaim Netcup RS 1000 G12 memiliki ~93K IOPS TERKONFIRMASI oleh benchmark YABS (Yet Another Bench Script).

Data VPSBenchmarks (Nov 2025):
4K Read IOPS: 48,200
4K Write IOPS: 48,200
Combined IOPS: 96,400

Real-world Capacity: Dengan 96K IOPS, Netcup dapat menangani 40-80+ concurrent Chrome instances (single instance memerlukan 500-2,000 IOPS).

4. Contabo IOPS Sangat Rendah

Data riset tentang Contabo (4K IOPS ~1.3-2.7K) adalah AKURAT. Komunitas melaporkan Contabo membatasi disk I/O ketat (~5 MB/s per arah = ~2.6K IOPS).

CPU Steal 30-80%

"Lottery-based" resource allocation tanpa predictability. Community verdict: "Ditch Contabo ASAP" untuk workload I/O intensive.

Koreksi yang Perlu Diperbaiki

1. Estimasi IOPS OVHcloud

Riset menyebut OVH VPS-1 ~10-15K IOPS. Data terbaru menunjukkan performa ~20K IOPS (read/write).

Riset Original: 10-15K IOPS
Data Aktual (2025): ~20K IOPS
Kesimpulan: Lebih baik, tapi masih jauh dari Netcup (93K)

2. CPU & Arsitektur OVH

OVH VPS-1 memiliki 4 vCPU, tapi performa per core rendah. Geekbench single-core ~1,165 (vs Netcup EPYC ~2,000+).

‚ö†Ô∏è "CPU 12 tahun lalu" - Kemungkinan OVH memakai Xeon lama untuk VPS-1. Core count tinggi (4) tapi kecepatan per core pas-pasan. Untuk OpenClaw yang butuh single-thread kuat, ini bisa jadi bottleneck.

3. ARM VPS (Hetzner CAX31) - Klarifikasi

Riset menyebut ARM = "not compatible". Perlu diluruskan: OpenClaw BISA jalan di ARM (bahkan ada yang install di Raspberry Pi).

‚úÖ Playwright

Support ARM64 native. Recommended untuk ARM deployment.

‚ùå Puppeteer

Limited support, perlu Chromium ARM manual atau x86 emulation.

Koreksi: ARM VPS compatible with adjustments, bukan tidak kompatibel. Butuh effort lebih untuk setup.
